    Integrate commit counting with fetch operations per UI-04 requirement:

    1. Add commit count command to cmds.go:
       ```go
       type commitCountsLoadedMsg struct {
           Counts git.BranchCommitCounts
       }

       func loadCommitCountsCmd(gitRunner git.Runner) tea.Cmd {
           return func() tea.Msg {
               // Get current branches from branchesPane
               branches := []string{"main", "master"} // Start with common defaults
               if current, err := gitRunner.GetCurrentBranch(); err == nil && current != "HEAD" {
                   branches = append(branches, current)
               }

               counts, err := gitRunner.GetBranchCommitCounts(branches)
               if err != nil {
                   logger.Get().Warn("failed to load commit counts: %v", err)
                   return commitCountsLoadedMsg{Counts: make(git.BranchCommitCounts)}
               }
               return commitCountsLoadedMsg{Counts: counts}
           }
       }
       ```

    2. Add commitCountsLoadedMsg handler to model.Update():
       ```go
       case commitCountsLoadedMsg:
           m.branchesPane.SetCommitCounts(msg.Counts)
           
           // Update status bar with summary
           totalAhead := 0
           for _, count := range msg.Counts {
               totalAhead += count.Ahead
           }
           if totalAhead > 0 {
               m.statusPane.SetNewCommitsAvailable(totalAhead)
           }
           return m, nil
       ```

    3. Trigger commit count loading after successful fetch:
       - In startupFetchResultMsg handler: add loadCommitCountsCmd when msg.Success && !msg.Skipped
       - In autoFetchResultMsg handler: add loadCommitCountsCmd when msg.Success && !msg.Skipped
       - Use tea.Batch to include with existing refresh commands

    4. Add status bar new commits indicator:
       - Extend StatusPane with SetNewCommitsAvailable(count int) method
       - Show in status bar: "repo → branch [3 new]" when commits available
       - Clear indicator when user switches to commits view

    Maintain existing fetch flow and don't disrupt current performance.
