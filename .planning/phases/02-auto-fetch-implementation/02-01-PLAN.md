  <behavior>
    - FetchBranches("origin", ["main", "feature"]) executes git fetch with branch refspecs
    - GetDefaultBranch("origin") returns "main" for repos with main as default
    - GetCurrentBranch() returns current branch name or "HEAD" for detached
    - All methods handle git command errors gracefully with proper error wrapping
  </behavior>
  <action>
    Create internal/git/fetch.go with three methods extending git.Runner:
    
    1. **FetchBranches(remote string, branches []string) error**
       - Uses git fetch with explicit refspecs: "branch:refs/remotes/remote/branch"
       - Handles empty branch list gracefully (no-op)
       - Uses NetworkTimeout for network operations
       - Returns wrapped errors with context
    
    2. **GetDefaultBranch(remote string) (string, error)**  
       - Uses git symbolic-ref refs/remotes/remote/HEAD for default branch detection
       - Falls back to git ls-remote --symref remote HEAD if symbolic-ref fails
       - Provides "main" as ultimate fallback if both methods fail
       - Handles authentication errors and network timeouts gracefully
    
    3. **GetCurrentBranch() (string, error)**
       - Uses git branch --show-current for current branch name
       - Returns "HEAD" for detached HEAD state
       - Handles git command errors with proper error context
    
    Follow GitZen patterns: Vietnamese comments, proper error wrapping with fmt.Errorf, use existing timeout constants.
  </action>
